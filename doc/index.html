Zen Cart v2 Japanese language pack documentation


Originally released on 11-11-2022, this language pack includes not only Japanese language files but although core files and database modifications adding possibility to use furigana and  products dimension anywhere in the cart system.

This allows use of Japanese Yamato express delivery, Nekoposu, Sagawa express delivery, Japan Post Yupack, LetterPack Lite, LetterPack Plus and EMS services. All these shipping modules files are included. Some payment and order total modules that complete shipping modules (c.o.d.) are although provided.

With this set you can use Zen Cart in Japan without having to create your own modules. Administration interface can be left in English if you have customers in Japan but are not Japanese or set to Japanese if you want a fully translated environment. You must choose at installation time because Zen Cart administration panel has only a limited support for internationalization.

Most of the translation is the work of members of Zen-Cart.jp which does not exist anymore. Ideas of adding furigana and Japanese shipping modules with shipping fees based on dimension are more than ten years old but they were redesigned for Zen Cart v2.0.0 and above. As in Japan reading of name's kanjis varies a lot from one region to another, furigana are useful if not necessary on the admin side.
Japan transport methods fees are generally based on parcel size with weight limits, even fixed rate shipping have some size limits. It is why products dimensions are necessary to provide an accurate quote.
All three major transport companies offer delivery time choice too and this option has been added too.

All these extra features involve modification of some core files and database, and it is why it is not a drop in plugin, nor a one click install. Although, for a new installation, when using ZC installer, it is close to it. But when it comes to upgrading an existing site, more steps are necessary and will be covered in this documentation.

Install

Before starting there are few  things to do or check.

First you need a space on a server to put all your files and it must be accessible through your web browser whatever it is local or on a remote server. That means you should have a virtual host set for your domain or sub-domain to access it through HTTP or better HTTPS. If you work on a remote server, an FTP client to upload your files will be useful.
Then you must set an empty MySQL database (Unicode collation) and a user with standard write-read-edit-delete rights on this database.
This pack is for Zen Cart 2.0.0 and above, please don't use too old versions for PHP and MySQL. You can check on Zen Cart site for exact requirement, but a good choice would be to use PHP 8.1 or newer and MySQL 8.xx.

Don't forget, this language pack is for Zen Cart 2.0.0 and above! It is not compatible with older versions. For ZC v1.5.8 and 1.5.8a, please use ZC Japanese Language Pack v1.3.x.

Finally, if you have a cart already installed, you must do a complete backup including database. For files a copy is easy, but for database you must export it in an SQL file you can import again if necessary. This is a very important step! In case of troubles, it will save you lot of time and more.

New cart installation

Extract Zen Cart v2.1.0 or newer you have just downloaded and copy all English ZC files in your site folder. Extract Zen Cart Japanese Language Pack too and copy all content of folder 'FILES_to_INSTALL' overwriting Zen Cart original files when necessary. Don't reverse order! Do Zen Cart first then language pack.

Now you should open your browser and in address bar type your site address using your real domain in place of 'yoursite' and http or https depending on your case:
https://yoursite.com/zc_install/ (or https://yoursite.com/ and click on 'Click here' link)

Zen Cart installer will start in Japanese (you can change to English) and do some system checks. If there is no big problem with system check, you can proceed to install like a normal Zen Cart by clicking on 'Clean install' button. After your first login in admin, installation is done!

Now you must add Japanese language to Zen Cart through 'admin>Localization>Languages'. This has to be done manually through admin interface as it sets everything necessary for a new language in Zen cart.

Name: Japanese
Code: ja
Image: icon.gif
Directory: japanese
Sort Order: 0

Set Japanese as default. Then import/run SQL queries in file 'orders_status_update.sql' (in 'sql' folder) to update order status table with Japanese.

Now you can go to configuration chapter to get ready for sale.

The only difference with standard Zen Cart install is that you must choose to install English admin sub-menus for Japanese or not. If you choose to install English admin  sub-menus, your admin interface will be in English even when you use it in Japanese. It will be partially translated and especially all part that customers might see like invoices and emails will be in Japanese. If you use it in English, everything will be in English.

Generally, if you are Japanese, you will keep Japanese admin menus. But if you are not, choosing English for what customers cannot see in your admin is a good option.

Updating an already installed English cart but still close to default setting

When installing over an existing Zen Cart 2.1.0 with no plugin that modify core files installed, it is nearly as easy as a new install but there are few points you need to check. Be careful that in this case of installation, Japanese Language Pack version and English ZC versions must correspond:

ZC v1.5.8-1.5.8a --> Japanese Language Pack v1.3.4
ZC v2.0.0-2.0.1 --> Japanese Language Pack v2.0.1
ZC v2.1.0 --> Japanese Language Pack v2.1.0

Best is to update ZC to latest English version then use latest Japanese Language Pack corresponding. If you don't respect this, database version will not be recognized correctly and database will not be updated to Japanese version.

First, after doing a backup of your installed cart, extract Zen Cart Japanese Language Pack and rename admin folder (in folder FILES_to_INSTALL) to the same name as your already installed ZC admin folder. If you want to use English admin version, copy admin-EN folder content (overwriting files) to admin folder before renaming. Then copy everything in folder FILES_to_INSTALL to your cart, overwriting original files.

Now you should open your browser and in address bar type your site address using your real domain in place of 'yoursite' and http or https depending on your case:
https://yoursite.com/zc_install/

Zen Cart installer will start in Japanese (you can change to English) and do some system checks. If there is no big problem with system check, you can proceed to install like a normal Zen Cart by clicking on 'Upgrade' button to upgrade your database. This is very important, don't click on 'Clean install' button, it will reset your database! If 'Upgrade' button is not visible, there is a problem. It means that your database version has not been recognized or it is a wrong version, or it is already upgraded. You must sort this out and fix it to be able to do the upgrade.

After clicking on 'Upgrade' button, on next screen you will have to enter your superuser credentials, but before you must check that on the list above there is at least one option like '2.1.0 to 2.1.0200' that correspond to your version. You can then proceed as a normal install.

Now you must add Japanese language to Zen Cart through 'admin>Localization>Languages'. This has to be done manually through admin interface as it sets everything necessary for a new language in Zen cart.

Installation on an English modified cart with plugins and other custom settings or on a Japanese version cart

If you already have Zen Cart with plugins installed, you might not be able to copy this Japanese language pack files over and expect everything to work. There are two possible situations, whether installed plugins have modified Zen Cart core files or not.

If none of the Zen cart original files have been modified, then it is easy, use the installer following instructions from precedent chapter above.

If core files have been modified (by plugins or other), there will be more steps and mainly two parts to deal with. One is files update and the other is database update. You can take care of these in any order, it is not important.

In Zen Cart documentation you will find some guide to update ZC that could be very useful.

Configuration

Most of specific configuration has been done at install but you still have to set your store location and postal code, define some geo-zones for your shipping modules and set these modules and payment ones too.
Please have a look at Zen Cart documentation, starting by here: https://docs.zen-cart.com/user/new_user_topics/

Modules setting

In this Language pack, shipping modules for main Japanese companies are included and you need to set them because shipping fees depends on your store location and eventually contract with shipping companies. Files to set are '_jpparcel.php', '_sagawa.php', '_yamato.php', '_yupack.php' in 'includes/classes/' folder. You need to check and adjust rates if necessary.
Quoting is based on three main arrays:
One for quote/shipping cost with one row for each price range and columns for size, '$a_pricerank'. This is where shipping cost is set.
Second one defines which range (in first one) to use depending on sending location and destination, '$a_dist_to_rank'. Coding system starts with two letters, each one corresponding to a prefecture/state group defined in third array ($a_zonemap). First letter is for sending location and second one for destination. Then follows a row number to use in first array for this location combination.

You must check and eventually set the array '$a_pricerank' to fit your situation. Unfortunately none of the shipping companies provide downloadable quoting tables but you can check online:
For Yamato : https://www.kuronekoyamato.co.jp/ytc/search/estimate/ichiran.html
For Sagawa : https://www.sagawa-exp.co.jp/send/fare/attention.html
For Japan Post YuPack : https://www.post.japanpost.jp/service/you_pack/charge/ichiran.html
For Japan Post overseas services : https://www.post.japanpost.jp/int/charge/list/

If you have some question go to support forum : https://www.zen-cart.com/showthread.php?229157-V158-Japanese-language-pack

Recommended plugins

Language/Currency Links in Header

This plugin should be in any multilingual Zen Cart. Having language switch in header is very useful and practical especially on mobile displays.

ZCA Bootstrap Template for Japanese Language Pack

This is a Japanese version of this nice template adapted to this language pack. It is not yet released in Zen Cart site plugin section but can be downloaded from GitHub here. It looks great on mobiles screen.

Postcode auto fill

This plugin was designed to help customers to fill create account or new address form. It simply auto fill state, city and sometime part of street address, when customer entered is postal code. It is not a necessary feature, but avoid errors like inconsistent address where postal code and address do not correspond. Drawback is that postal code databases are big and plugin had to be separated in two parts. Importing big files in MySQL with phpMyAdmin could be problem too if PHP and MySQL are not set properly.

Image Handler 5 (Zen Cart 1.5.7+)

This plugin enhances Zen Cart images by adding a handler to offer automatic resizing of images, caching thumbnails, and more. You can upload multiple sizes of an image from the Admin image handler page.

Uninstall

First, you must delete Japanese language from Zen Cart Admin menu 'Localization>Languages'.
Delete all 'newly' added files listed in 'List of new files to copy'.
Use your backup to copy back your precedent version for all merged/replaced files listed in 'List of files to merge'.
Import SQL file 'uninstall.sql' which is in 'sql' folder of the language pack. It will remove all extra fields that have been added to database and reset to default settings that were modified at install time. Be careful that any information added in new fields created by this module will be lost.
Be aware too that default settings might not be what you want, you must checks these and adjust as you want them to be. Mainly these setting are:
Default language resets to English.
Default currency resets to USD.
Store country resets to United States.

Bonus content

You will find some language files translation for few plugins in folder '...\optional plugins'.

Zen Cart v2 Japanese language pack 説明書


この日本語パックには、言語ファイルだけでなく、コア ファイルとデータベースの変更が含まれており、ふりがな、商品寸法、バーコードを使用する可能性が追加されています。
これにより、日本のヤマト便、ネコポス、佐川急便、日本郵便ゆうパック、レターパック ライト、レターパック プラス、および EMS サービスを利用できます。配送、支払い、および注文の合計モジュール ファイルが含まれています。
このセットにより、日本で Zen Cart を使用できるようになります。日本に顧客がいて日本人ではない場合、管理インターフェースを英語のままにしておくか、完全に翻訳された環境が必要な場合は日本語に設定でき ます。インストール時に選択する必要があります。

翻訳のほとんどは、もう存在しない Zen-Cart.jp のメンバーの作品です。
ふりがなと日本の配送モジュールを追加し、寸法に基づいた配送料を設定するというアイデアは 10 年以上前のものですが、Zen Cart v2.0.0 で再設計されました。
日本では名前の漢字の読み方が地域によって大きく異なるため、管理者側でもフリガナは非常に便利です。
日本の配送方法の料金は、荷物のサイズに基づいており、重量制限もあります。固定料金の配送でもサイズ制限がありますので、正確な見積もりを提供するため に商 品の寸法が必要です。
そして、三つの主要な運送会社はすべて、配達時間の選択も提供しています。これらのパラメータがデータベースに追加されたのはそのためです。もちろん、い くつ かのコア ファイルが大幅に変更されています。
バーコードだけは実は日本語版とは関係ありませんが、コア ファイルの変更が既に存在するため、これを追加するのにそれほど労力は必要ありませんでした。

インストール

開始する前に、やるべきことや確認すべきことがいくつかあります。

まず、ローカルまたはリモート サーバー上にファイルを配置するためのスペースが必要であり、Web ブラウザからアクセスできる必要があります。つまり、HTTP またはより優れた HTTPS 経由でアクセスするには、ドメインまたはサブドメインに仮想ホストを設定する必要があります。リモート サーバーを使用する場合は、ファイルをアップロードするための FTP クライアントが便利です。
次に、空の MySQL データベース (Unicode 照合順序) と、このデータベースに対する標準の書き込み、読み取り、編集、削除の権限を持つユーザーを設定する必要があります。
このパックは Zen Cart 2.0.0 以降を対象としています。PHP および MySQL には古すぎるバージョンを使用しないでください。正確な要件については、Zen Cart サイトでご確認ください。ただし、PHP 8.1 以降を使用することをお勧めします。

この言語パックは Zen Cart 2.0.0 以降用であることを忘れないでください。 古いバージョンとは互換性 がありません。ZC v1.5.8 および 1.5.8a の場合は、ZC 日本語言語パック v1.3.x を使用してください。

最後に、カートがすでにインストールされている場合は、データベースを含む完 全なバックアップを実行する必要があります。ファイルの場合、コピーは簡単ですが、データベースの場合は、必要に応じて再度インポート できる SQL ファイルにエクスポートする必要があります。これは非常に重要なステップです。 トラブルが発生した場合でも、時間を大幅に節約できます。詳細な説明を表示するには、以下のメニューから選択してください。

特定の構成のほとんどはインストール時に行われていますが、店舗の場所と郵便番号を設定し、配送モジュール用にいくつ かの 地域ゾーンを定義し、これらのモジュールと支払いモジュールも設定する必要があります。
ここから始まる Zen Cart のドキュメントをご覧ください（英語）： https://docs.zen-cart.com/user/new_user_topics/

モジュールの設定

この言語パックには日本の主要企業の配送モジュールが含まれており、配送料は店舗の所在地や最終的に配送会社との契約 に応 じて異なるため、設定する必要があります。 設定するファイルは「includes/classes/」フォルダ内の「_jpparcel.php」、「_sagawa.php」、「_yamato.php」、 「_yupack.php」です。必要に応じて料金を確認し、調整する必要があります。
クォートは 3 つの主要な配列に基づいています：
見積もり/送料の「$a_pricerank」。価格帯ごとに 1 行、サイズの列が含まれます。送料の設定はこちらで行います。
2 番目の '$a_dist_to_rank' は、送信場所と宛先に応じて (最初の範囲内の) どの範囲を使用するかを定義します。コーディング システムは 2 つの文字で始まり、各文字は 3 番目の配列 ($a_zonemap) で定義された都道府県グループに対応します。1 番目の文字は送信場所、2 番目の文字は送信先を表します。次に、この位置の組み合わせの最初の配列で使用する行番号が続きます。

これら 2 つの配列 ('$a_pricerank' と '$a_dist_to_rank') を確認し、最終的に状況に合わせて設定する必要があります。 残念ながら、ダウンロード可能な見積表を提供している配送会社はありませんが、オンラインで確認できます。
ヤマト宅急便： https://www.kuronekoyamato.co.jp/ytc/search/estimate/ichiran.html
佐川急便発送： https://www.sagawa-exp.co.jp/send/fare/attention.html
日本郵便ゆうパック： https://www.post.japanpost.jp/service/you_pack/charge/ichiran.html
日本郵便の海外サービス： https://www.post.japanpost.jp/int/charge/list/

質問がある場合は、サポート フォーラムにアクセスしてください。 https://www.zen-cart.com/showthread.php?229157-V158-Japanese-language-pack

おすすめのプラグイン

Language/Currency Links in Header

このプラグインは、多言語の Zen Cart に含まれている必要があります。 ヘッダーに言語切り替えがあると、特にモバイル ディスプレイで非常に便利で実用的です。

ZCA Bootstrap Template for Japanese Language Pack

これは、この言語パックに適合したこの素晴らしいテンプレートの日本語版です。 Zen Cart サイトのプラグイン セクションではまだリリースされていませんが、GitHub からダウンロードで きます。 携帯電話の画面では見栄えがします。

Postcode auto fill

このプラグインは、顧客がアカウント作成フォームまたは新しいアドレス フォームに記入できるように設計されました。顧客が郵便番号を入力すると、州、都市、および番地の一部が自動的に入力されます。必須の機能ではありませんが、郵便番号と住 所が一致しない住所の不一致などのエラーを回避します。欠点は、郵便番号データベースが大きく、プラグインを 2 つの部分に分割する必要があることです。PHP と MySQL が適切に設定されていない場合、phpMyAdmin を使用して MySQL に大きなファイルをインポートする場合にも問題が発生する可能性があります。

Image Handler 5 (Zen Cart 1.5.7+)

このプラグインは、画像の自動サイズ変更、サムネイルのキャッシュなどを提供するハンドラーを追加することにより、 Zen Cart 画像を強化します。 管理者イメージ ハンドラー ページから複数のサイズのイメージをアップロードできます。

Uninstall

まず、Zen Cart 管理メニューの「ローカリゼーション > 言語」から日本語を削除する必要があります。
「コ ピーする新しいファイルのリスト」にリストされている新しく追加されたファイルをすべて削除します。
バックアップを使用して、「結 合するファイルのリスト」にリストされているすべての結合/置換されたファイルの前のバージョ ンをコピーバックします。
言語パックの「sql」フォルダーにある SQL ファイル「uninstall.sql」をインポートします。 データベースに追加された余分なフィールドがすべて削除され、インストール時に変更されたデフォルト設定にリセットされます。このモジュールによって作成された新しい フィールドに追加された情報はすべて失われることに注意してください。
デフォルト設定が希望どおりでない可能性があることにも注意してください。これらを確認し、希望どおりに調整する必要があります。主に次の設定があり ま す。
デフォルトの言語が英語にリセットされました。
デフォルトの通貨が米ドルにリセットされました。
購入国を米国にリセットします。
