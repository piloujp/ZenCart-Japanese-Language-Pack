Zen Cart v1.5.8 Japanese language pack documentation


Originally released on 11-11-2022, this language pack includes not only Japanese language files but although core files and database modifications adding possibility to use furigana and  products dimension anywhere in the cart system.

This allows use of Japanese Yamato express delivery, Nekoposu, Sagawa express delivery, Japan Post Yupack, LetterPack Lite, LetterPack Plus, International ePackets and EMS services. All these shipping modules files are included. Some payment and order total modules that complete shipping modules (c.o.d.) are although provided.

With this set you can use Zen Cart in Japan without having to create your own modules. Administration interface can be left in English if you have customers in Japan but are not Japanese or set to Japanese if you want a fully translated environment. You must choose at installation time because Zen Cart administration panel has only a limited support for internationalization.

Most of the translation is the work of members of Zen-Cart.jp which does not exist anymore. Ideas of adding furigana and Japanese shipping modules with shipping fees based on dimension are more than ten years old but they were redesigned for Zen Cart v1.5.8. As in Japan reading of name's kanjis varies a lot from one region to another, furigana are useful if not necessary on the admin side.
Japan transport methods fees are generally based on parcel size with weight limits, even fixed rate shipping have some size limits. It is why products dimensions are necessary to provide an accurate quote.
All three major transport companies offer delivery time choice too and this option has been added too.
Only barcode is not really link to Japanese version but as there were already core files modification, adding this did not required much more efforts.

All these extra features involve modification of some core files and database, and it is why it is not a drop in plugin, nor a one click install. Although, for a new installation using ZC installer available from release v1.3.0, it is close to it. But when it comes to upgrading an existing site, more steps are necessary and will be covered in this documentation.

Install

Before starting there are few  things to do or check.

First you need a space on a server to put all your files and it must be accessible through your web browser whatever it is local or on a remote server. That means you should have a virtual host set for your domain or sub-domain to access it through HTTP or better HTTPS. If you work on a remote server, an FTP client to upload your files will be useful.
Then you must set an empty MySQL database (Unicode collation) and a user with standard write-read-edit-delete rights on this database.
This pack is for Zen Cart 1.5.8 and above, please don't use too old versions for PHP and MySQL. You can check on Zen Cart site for exact requirement, but a good choice would be to use PHP 8.1 or newer and MySQL 5.7 will do but 8.0x is better. MySQL 8.0 has some new collation and properties for Japanese that are interesting. For backward compatibility reasons there are not used in this plugin for now, but it might change in the future.

Don't forget, this language pack is for Zen Cart 1.5.8a and above! It works with ZC 1.5.8 but won't with older versions.

Finally, if you have a cart already installed, you must do a complete backup including database. For files a copy is easy, but for database you must export it in an SQL file you can import again if necessary. This is a very important step! In case of troubles, it will save you lot of time and more. Please, choose from below menu to get detailed explanations:

New cart installation

Extract Zen Cart v1.5.8a or newer you have just downloaded and copy all files in your site folder. Extract Zen Cart v1.5.8 Japanese Language Pack too and copy admin, email, images, includes and zc_install folders overwriting Zen Cart original files when necessary. Don't reverse order! Do Zen Cart first then language pack.

Now you should open your browser and in address bar type your site address using your real domain in place of 'yoursite' and http or https depending on your case:
https://yoursite.com/zc_install/

Zen Cart installer will start in Japanese (you can change to English) and do some system checks. If there is no big problem with system check, you can proceed to install like a normal Zen Cart by clicking on 'Clean install' button. After your first login in admin, installation is done! Go to configuration chapter to get ready for sale.

The only difference with standard Zen Cart install is that you must choose to install Japanese admin  sub-menus or not. Most of sub-menus in Zen Cart admin are hard coded in database and cannot be changed on the fly easily. They won't change even if you switch language with language selector. If you choose to install these, you cart will be fully Japanese. If not, your admin interface will be in English but when you use it in Japanese, it will be partially translated and especially all part that customers might see like invoices and emails will be in Japanese. If you use it in English, everything will be in English.

Generally, if you are Japanese, you will choose Japanese admin menus. But if you are not, keeping in English what customers cannot see in your admin is a good option. This language setting can be changed manually afterward, see configuration chapter for that.

Already installed cart but still close to default setting

When installing over an existing Zen Cart 1.5.8 or 1.58a with no plugin that modify core files installed, it is nearly as easy as a new install but there are few points you need to check.

First, after doing a backup of your installed cart, extract Zen Cart v1.5.8 Japanese Language Pack and rename admin folder to the same name as your already installed admin folder. Then copy everything in folders admin (renamed to your real admin folder name), email, images, includes and zc_install to your cart overwriting original files.

Now you should open your browser and in address bar type your site address using your real domain in place of 'yoursite' and http or https depending on your case:
https://yoursite.com/zc_install/

Zen Cart installer will start in Japanese (you can change to English) and do some system checks. If there is no big problem with system check, you can proceed to install like a normal Zen Cart by clicking on 'Upgrade' button to upgrade your database. This is very important, don't click on 'Clean install' button, it will erase your database! If 'Upgrade' button is not visible, there is a problem. It means that your database version has not been recognized or it is a wrong version, or it is already upgraded. You must sort this out and fix it to be able to do the upgrade.

After clicking on 'Upgrade' button, on next screen you will have to enter your superuser credentials but before you must check that on the list above there is only one option that is checked, and it is '1.5.8 to 1.5.8130'. You can then proceed as a normal install.

When upgrading database there are few settings that are done like setting Japanese as default language, Japanese Yen as default currency, adding new tax class and rate... You should check in admin that these setting are like you want them. For example, if you have more than two languages installed, you might want to use Japanese but not as default language. In this case, you must set this as you want in admin afterward. If all is good, you can go to configuration chapter.

Installation on a modified cart with plugins and other custom settings

If you already have Zen Cart 1.5.8 with plugins installed, you might not be able to copy this Japanese language pack files over and expect everything to work. There are two possible situations, whether installed plugins have modified Zen Cart 1.5.8 core files or not.

If none of the Zen cart original files have been modified, then it is easy, use the installer following instructions from precedent chapter above.

If core files have been modified, there will be more steps and mainly two parts to deal with. One is files upgrade and the other is database update. You can take care of these in any order, it is not important.

Database update:

To do this step a good tool will make things easier and faster, something like 'phpMyAdmin' is perfect for this job. Zen Cart admin can be used too for importing SQL files but it won't be easy to check and see what you did, and you might run into trouble too as we modify Zen Cart admin too.

There is a folder named 'sql' where you will find the two SQL files we need to update Zen Cart database:

'mysql_japanese_install.sql'

This file contains database modifications and configurations for this Japanese Language Pack. It includes new fields for furigana, phone, fax number and delivery time in tables orders, address_book and customers then fields for dimensions and barcode in table products. There are some added records like new order status, address format and Japanese zones in kanji and romaji.
This is the only file you really NEED to INSTALL for everything to work. Other SQL files are optional. Import this file in your cart's database using phpMyAdmin import function and your are done!

If you want a fully Japanese admin side, then import next file as well, it will translate admin menus kept in database into Japanese:

'mysql_japanese_admin_menu.sql'

Configuration

Most of specific configuration has been done at install but you still have to set your store location and postal code, define some geo-zones for your shipping modules and set these modules and payment ones too.
Please have a look at Zen Cart documentation, starting by here: https://docs.zen-cart.com/user/new_user_topics/

Modules setting

In this Language pack, shipping modules for main Japanese companies are included and you need to set them because shipping fees depends on your store location and eventually contract with shipping companies. Files to set are '_jpparcel.php', '_sagawa.php', '_yamato.php', '_yupack.php' in 'includes/classes/' folder. You need to check and adjust rates if necessary.
For YuPack you might need to set a new category depending on where your store is located. For other you can uncomment what fits your need.
Quoting is based on three main arrays:
One for quote/shipping cost with one row for each price range and columns for size, '$a_pricerank'. This is where shipping cost is set.
Second one defines which range (in first one) to use depending on sending location and destination, '$a_dist_to_rank'. Coding system starts with two letters, each one corresponding to a prefecture/state group defined in third array ($a_zonemap), first one for sending location and second one for destination. Then follows a row number to use in first array for this location combination.

You must check and eventually set these two arrays ('$a_pricerank' and '$a_dist_to_rank') to fit your situation. Unfortunately none of the shipping companies provide downloadable quoting tables but you can check online:
For Yamato : https://www.kuronekoyamato.co.jp/ytc/search/estimate/ichiran.html
For Sagawa : https://www.sagawa-exp.co.jp/send/fare/attention.html
For Japan Post YuPack : https://www.post.japanpost.jp/service/you_pack/charge/ichiran.html
For Japan Post overseas services : https://www.post.japanpost.jp/int/charge/list/
For Japan Post International ePackets : https://www.post.japanpost.jp/int/service/epacket.html

If you have some question go to support forum : https://www.zen-cart.com/showthread.php?229157-V158-Japanese-language-pack

Admin language:

Admin language in Zen Cart is peculiar, there are two parts, one that use language files and another that use database to store language relative data. Language files can easily be switch from one language to another but with database it is more complicated and not as safe. In Zen Cart, Database part is not switchable 'on the fly' but it can be done by importing an SQL file.
Using this two files in 'sql' folder you can switch between English and Japanese sub-menus:

'mysql_japanese_admin_menu.sql'
'mysql_english_admin_menu.sql'

Recommended plugins

Language/Currency Links in Header

This plugin should be in any multilingual Zen Cart. Having language switch in header is very useful and practical especially on mobile displays.

ZCA Bootstrap Template for Japanese Language Pack

This is a Japanese version of this nice template adapted to this language pack. It is not yet released in Zen Cart site plugin section but can be downloaded from GitHub here. It looks great on mobiles screen.

Postcode auto fill

This plugin was designed to help customers to fill create account or new address form. It simply auto fill state, city and sometime part of street address, when customer entered is postal code. It is not a necessary feature, but avoid errors like inconsistent address where postal code and address do not correspond. Drawback is that postal code databases are big and plugin had to be separated in two parts. Importing big files in MySQL with phpMyAdmin could be problem too if PHP and MySQL are not set properly.

Image Handler 5 (Zen Cart 1.5.7+)

This plugin enhances Zen Cart images by adding a handler to offer automatic resizing of images, caching thumbnails, and more. You can upload multiple sizes of an image from the Admin image handler page.

Uninstall

Delete all 'newly' added files listed in 'List of new files to copy'.
Use your backup to copy back your precedent version for all merged/replaced files listed in 'List of files to merge'.
Import SQL file 'uninstall.sql' which is in 'sql' folder of the language pack. It will remove all extra fields that have been added to database and reset to default settings that were modified at install time. Be careful that any information added in new fields like products sizes will be lost.
Be aware too that default settings might not be what you want, you must checks these and adjust as you want them to be. Mainly these setting are:
Default language reset to English.
Default currency reset to USD.
Store country reset to United States.

Zen Cart v1.5.8 Japanese language pack 説明書


この日本語パックには、言語ファイルだけでなく、コア ファイルとデータベースの変更が含まれており、ふりがな、商品寸法、バーコードを使用する可能性が追加されています。
これにより、日本のヤマト便、ネコポス、佐川急便、日本郵便ゆうパック、レターパック ライト、レターパック プラス、および EMS サービスを利用できます。配送、支払い、および注文の合計モジュール ファイルが含まれています。
このセットにより、日本で Zen Cart を使用できるようになります。日本に顧客がいて日本人ではない場合、管理インターフェースを英語のままにしておくか、完全に翻訳された環境が必要な場合は日本語に設定でき ます。インストール時に選択する必要があります。

翻訳のほとんどは、もう存在しない Zen-Cart.jp のメンバーの作品です。
ふりがなと日本の配送モジュールを追加し、寸法に基づいた配送料を設定するというアイデアは 10 年以上前のものですが、Zen Cart v1.5.8 で再設計されました。
日本では名前の漢字の読み方が地域によって大きく異なるため、管理者側でもフリガナは非常に便利です。
日本の配送方法の料金は、荷物のサイズに基づいており、重量制限もあります。固定料金の配送でもサイズ制限がありますので、正確な見積もりを提供するため に商 品の寸法が必要です。
そして、三つの主要な運送会社はすべて、配達時間の選択も提供しています。これらのパラメータがデータベースに追加されたのはそのためです。もちろん、い くつ かのコア ファイルが大幅に変更されています。
バーコードだけは実は日本語版とは関係ありませんが、コア ファイルの変更が既に存在するため、これを追加するのにそれほど労力は必要ありませんでした。

インストール

開始する前に、やるべきことや確認すべきことがいくつかあります。

まず、ローカルまたはリモート サーバー上にファイルを配置するためのスペースが必要であり、Web ブラウザからアクセスできる必要があります。つまり、HTTP またはより優れた HTTPS 経由でアクセスするには、ドメインまたはサブドメインに仮想ホストを設定する必要があります。リモート サーバーを使用する場合は、ファイルをアップロードするための FTP クライアントが便利です。
次に、空の MySQL データベース (Unicode 照合順序) と、このデータベースに対する標準の書き込み、読み取り、編集、削除の権限を持つユーザーを設定する必要があります。
このパックは Zen Cart 1.5.8 以降を対象としています。PHP および MySQL には古すぎるバージョンを使用しないでください。正確な要件については、Zen Cart サイトでご確認ください。ただし、PHP 8.1 以降を使用することをお勧めします。MySQL 5.7 でも十分ですが、8.0x の方が優れています。MySQL 8.0 には、日本語用の興味深い新しい照合順序とプロパティがいくつかあります。下位互換性の理由から、現時点ではこのプラグインでは使用されていませんが、将来的には変更され る可能性があります。

この言語パックは Zen Cart 1.5.8a 以降用であることを忘れないでください。 ZC 1.5.8 では動作しますが、古いバージョンでは動作しません。

最後に、カートがすでにインストールされている場合は、データベースを含む完 全なバックアップを実行する必要があります。ファイルの場合、コピーは簡単ですが、データベースの場合は、必要に応じて再度インポート できる SQL ファイルにエクスポートする必要があります。これは非常に重要なステップです。 トラブルが発生した場合でも、時間を大幅に節約できます。詳細な説明を表示するには、以下のメニューから選択してください。

初め てのインストール

ダウンロードした Zen Cart v1.5.8a 以降を抽出し、サイト フォルダー内のすべてのファイルをコピーします。Zen Cart v1.5.8 日本語言語パックも抽出し、必要に応じて admin、email、images、includes、および zc_install フォルダーをコピーして、Zen Cart の元のファイルを上書きします。順序を逆にしないでください！ 最初に Zen Cart を実行し、次に言語パックを実行します。

次に、ブラウザを開き、アドレス バーに、「yoursite」の代わりに実際のドメインを使用し、場合に応じて http または https を使用してサイト アドレスを入力する必要があります。
https://yoursite.com/zc_install/

Zen Cartインストーラーは日本語で起動し (英語に変更できます)、いくつかのシステム チェックを行います。システムチェックで大きな問題がなければ、「クリーンインストール」ボタンをクリックして通常のZen　Cartと同様にインストールを進めてくださ い。管理者に初めてログインすると、インストールが完了します。販売の準備をするには、設定の章に進んでください。

標準の Zen Cart インストールとの唯一の違いは、日本語の管理サブメニューをインストールするかどうかを選択する必要があることです。Zen Cart 管理者のサブメニューのほとんどはデータベースにハードコーディングされており、その場で簡単に変更することはできません。言語セレクターで言語を切り替えても変更されま せん。 日本語サブメニューのインストールを選択すると、カートは完全に日本語化されます。そうでない場合、管理インターフェースは英語になりますが、日本語で使用すると部分的に 翻訳され、特に請求書や電子メールなど、顧客が目にする可能性のあるすべての部分が日本語になります。英語で使用するとすべて英語になります。

通常、あなたが日本人の場合は、日本語の管理者メニューを選択しますしかし、そうでない場合は、管理画面で顧客に表示 されない部分を英語のままにしておくのが良い選択肢です。この言語設定は後で手動で変更できます。詳細については、設定の章を参照してください。

クォートは 3 つの主要な配列に基づいています：
見積もり/送料の「$a_pricerank」。価格帯ごとに 1 行、サイズの列が含まれます。送料の設定はこちらで行います。

管理言語

Zen Cart の管理言語は独特で、言語ファイルを使用する部分と、言語関連データを保存するデータベースを使用する部分の 2 つの部分があります。言語ファイルはある言語から別の言語に簡単に切り替えることができますが、データベースの場合はより複雑で安全ではありません。Zen Cart では、データベース部分の言語を即座に切り替えることはできませんが、SQL ファイルをインポートすることで切り替えることができます。

'mysql_japanese_admin_menu.sql'
'mysql_english_admin_menu.sql'

Uninstall
